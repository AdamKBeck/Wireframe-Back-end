Operations

Visibility Name Description
public elements(): List<Elements>

Returns the list of elements on the canvas. This is a List, as order matters. The list is ordered by how leftmost and topmost the element is on the canvas.

FIrst, an element closer to the top of the canvas has more priority in the list than an element closer to the bottom. Then, an element closer to the left has more priority than an element closer to the right of the canvas.
public width(): Int
public height(): Int
public add(element: Element)

The barricade class is called, and after iterating through each element in "THIS" element's group, add it to the canvas. Otherwise, do not add it to the canvas, but keep the element initialized.

Note: It's not correct to assume that as we add an element, it's not already part of a group. That's semantic coupling, among other things.

To add an element, insert it into the correct position in the list of elements based on it's x and y coordinate (see elements() for more detail).
